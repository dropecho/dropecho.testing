# Development Guide

## Building

```bash
# Build library (generates docs.xml)
haxe build.hxml

# Generate documentation
haxe targets/docs.hxml
```

## Running Tests

```bash
# Via haxelib run (reads test.hxml and .dropecho.testing.json)
haxelib run dropecho.testing

# Or directly if test.hxml specifies a single target
haxe --main dropecho.testing.AutoTest test.hxml
```

## Adding Support for a New Test Framework

1. Add a value to the `DetectedTestLib` enum in `DetectTestLib.hx`
2. Add detection logic in `DetectTestLib.detectTestLib()` using `Context.getType()`
3. Create a new `*IncludeMacros.hx` file modeled after `BuddyIncludeMacros.hx` or `UTestIncludeMacros.hx`
4. Add a dispatch branch in `IncludeTestsMacro.buildTests()`

## Adding a New Compilation Target

In `tools/Run.hx`:
1. Add the target name to `TargetType` abstract enum
2. Add the runner CLI command to `TargetRunner` abstract enum
3. Add the mapping in `GetTargetRunner()` (or equivalent switch)

## Instrumentation Integration

To add a new instrumentation hook in generated test runners:
- Edit `BuddyIncludeMacros.hx` and `UTestIncludeMacros.hx`
- Wrap new calls in `#if instrument ... #end`
- Keep symmetry between Buddy and UTest implementations

## Project Config Setup

```bash
# Create default .dropecho.testing.json in a project using this library
haxelib run dropecho.testing setup
```

## CI

CI runs via GitHub Actions, delegating to the shared workflow:
`.github/workflows/haxe-ci.yml` at `dropecho/.github`

Triggers: push to `main`, manual dispatch (`workflow_dispatch`).

## Release

Releases are fully automated via semantic-release on merge to `main`. Commit messages must follow the Angular convention:
- `feat:` → minor version bump
- `fix:` → patch bump
- `BREAKING CHANGE:` → major bump
- `chore(release):` commits are generated by semantic-release itself (skip CI)
